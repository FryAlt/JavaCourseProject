## Функциональность:

### Аутентификация и авторизация

Точка входа: /login (реализована с использованием Spring Security Form Login).

После успешной аутентификации пользователь перенаправляется на /books (доступно без аутентификации).

Доступ к административным ресурсам (/admin/**) ограничен: разрешён только сущностям, обладающим ролью ROLE_ADMIN.

### Страницы для авторизованных пользователей

После входа в систему пользователь может использовать следующие страницы:

- /books — отображает список всех книг. Поддерживается фильтрация (например, по жанру или автору) через параметры URL.
- /books/{id} — показывает подробную информацию о конкретной книге: название, автор, жанр, ISBN и цена.
- /cart — страница корзины.
- /favorites - страница избранных книг.
- /profile — личный кабинет: здесь можно посмотреть профиль, историю заказов и корзину.

Все эти страницы доступны только после успешной аутентификации.

### Управление состоянием корзины

Корзина пользователя персистентная и хранится в таблице cart_items базы данных.

Каждая запись в cart_items связана с user_id и book_id, содержит количество и метаданные (например, дата добавления).

При повторной аутентификации пользователя система восстанавливает его корзину из БД — состояние сохраняется между сессиями.

Неавторизованные пользователи не могут взаимодействовать с корзиной; попытка доступа к /cart перенаправляет их на /login.

### Административные страницы
Администраторы (пользователи с ролью ROLE_ADMIN) могут управлять каталогом книг через следующие страницы:

- /admin/books — общий список книг с возможностью просмотра.
- /admin/books/new — форма для добавления новой книги. После отправки данные сохраняются в базу.
- /admin/books/{id}/edit — форма редактирования существующей книги.
- Удаление книги выполняется отправкой POST-запроса на адрес /admin/books/{id}/delete (обычно через кнопку в интерфейсе).
